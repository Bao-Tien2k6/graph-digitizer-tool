Top-level layout

The app is a single-page Streamlit layout with a title row and a two-column body (st.columns([4, 6])). State machine: upload → calibrate → detect → edit → confirm → download → next image. Session keys are listed at the bottom of this page.

Left column (40%)

  • Upload widget — multi-file (render_upload_widget)
  • Batch indicator — “Image N of M”
  • Calibration step — axes overlay, tick editors, scale metrics, confirm button (render_calibration_step)
  • Detection overlay preview — after Stage 3
  • Next image button — batch advance

Right column (60%)

  • Image preview before detection (render_preview)
  • Data editor after detection — one expander per series, editable x/y, delete checkbox, Confirm button (render_data_editor)
  • Final overlay after confirmation

Full-width footer

  • Download Results — Final CSV + Overlay PNG (render_download_buttons)
  • Saved-to-disk caption — shows result_dir

Session state keys

  • uploaded_img — current BGR image (np.ndarray)
  • axes_info — Stage 2 output, mutated by tick editor
  • global_ocr_results — cached PaddleOCR output
  • routing_result — Stage 3 chart-type routing
  • transform_result — Stage 5 points + overlay paths
  • calibration_done, points_confirmed — step flags
  • queue_keys, queue_idx, processed_key — batch queue state
  • upload_name, result_dir — output naming
  • tick_X_i, tick_Y_i, editor_series_{sid} — widget keys

Legend

  • Hatched grey blocks = image preview area (Streamlit st.image).
  • Red buttons = type="primary" in Streamlit.
  • Blue notes = explanatory annotations, not part of the rendered UI.
